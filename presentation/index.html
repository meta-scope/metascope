Metascope – Data NFTs Rendered Human

Metascope

Data NFTs Rendered Human

Problem

  • It’s often necessary to inspect an NFT’s metadata.
  • On Etherscan it’s a laborious process requiring 6 steps.
  • Some values are off-chain, involving an http request.
  • Because the data is intended for machines it’s obscure for humans to read.

Complexity

  • PFP NFTs have a standard metadata schema that is known and used by NFT markets and wallets.
  • Data NFTs have custom metadata fields, often unique to their type (e.g. Uniswap V3, Superfluid, Ocean Protocol, token gating, etc.)
  • These types need to be recognized and rendered.

Solution

  • Metascope accepts an address of an NFT as input and returns its metadata in human readable form.
  • NFT types with a known schema are recognized and their custom fields are rendered.
  • Dates and other hexadecimal values are converted to a human readable format.

Technique

  • Metascope uses APIs such as AirStack for retrieving the standard metadata, falling back to Etherscan for unknown fields.
  • The frontend uses React and Something, with Netlify for hosting.
